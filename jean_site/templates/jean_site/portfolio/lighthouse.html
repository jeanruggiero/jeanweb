A Framework for Test-Driven Algorithms

I'm taking an algorithms in C class this semester, and by the second assignment, I already needed a better way to test my code on the command line. I'm currently focused on being disciplined about test-driven development, so I was looking to make testing fast and easy. I started building some unit testing infrastructure and using it to run my tests so that I could quickly tell which tests failed. I realized what I had built may be useful for my classmates and anyone else developing small C programs, so I spun it off into a header file, added some documentation and examples of its usage, and pushed it up to GitHub for anyone to use.

What's Next?

I hope to improve on this project throughout the semester and I've invited my classmates to contribute. If it becomes portable and/or useful enough, I hope to turn it into my first open source project.
